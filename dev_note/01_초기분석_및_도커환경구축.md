## 2. Docker 환경 구축
온프레미스 배포를 위한 Docker 컨테이너 환경을 구성했습니다.

### 주요 변경 사항
1.  **`Dockerfile` 생성**:
    - Multi-stage build 적용 (deps -> builder -> runner)으로 이미지 크기 최적화.
    - `node:20-alpine` 기반.
2.  **`docker-compose.yml` 생성**:
    - 서비스명: `web`
    - 포트 맵핑: `3000:3000`
3.  **`next.config.mjs` 수정**:
    - `output: 'standalone'` 설정 추가 (Docker 빌드 최적화).
4.  **`package.json` 수정**:
    - `packageManager`: `pnpm@10.22.0` 명시 (빌드 안정성 확보).

## 3. 트러블슈팅 (Troubleshooting)
구축 과정에서 발생한 문제와 해결 방법입니다.

### 3.1. ThemeProvider SSR 이슈
- **문제**: `pnpm run build` 시 `/dashboard/settings` 페이지에서 `Error: useTheme must be used within a ThemeProvider` 에러 발생.
- **원인**: `components/theme-provider.tsx`에서 `mounted` 상태가 아닐 때 `children`만 반환하여, SSR 시점에 Context가 제공되지 않음.
- **해결**: `mounted` 여부와 관계없이 항상 `ThemeContext.Provider`를 렌더링하도록 수정.

### 3.2. Lockfile 불일치 및 빌드 오류
- **문제**: Docker 빌드 중 `pnpm-lock.yaml` 관련 오류 및 의존성 설치 실패.
- **해결**:
    - 로컬에서 `pnpm install`을 실행하여 `pnpm-lock.yaml` 갱신.
    - `package.json`에 `packageManager` 필드를 추가하여 Corepack이 올바른 pnpm 버전을 사용하도록 강제.
